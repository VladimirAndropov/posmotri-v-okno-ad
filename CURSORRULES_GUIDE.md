# Руководство по .cursorrules

## Что такое .cursorrules?

`.cursorrules` — это файл конфигурации для Cursor IDE, который позволяет задавать правила и инструкции для AI-ассистента. Эти правила помогают AI лучше понимать контекст вашего проекта и следовать вашим предпочтениям в коде.

## Где создать файл?

Создайте файл `.cursorrules` в **корне вашего проекта** (там же, где `index.html`).

## Формат файла

Файл `.cursorrules` — это обычный текстовый файл. Вы можете писать правила в свободной форме, как инструкции для AI.

## Основные типы правил

### 1. **Стиль кода и соглашения**

```
- Используй BEM методологию для именования CSS классов
- Всегда используй одинарные кавычки в JavaScript
- Используй const/let вместо var
- Добавляй комментарии на русском языке
```

### 2. **Структура проекта**

```
- CSS файлы находятся в папке styles/
- JavaScript файлы в папке scripts/
- Используй модульную структуру CSS (отдельные файлы для компонентов)
```

### 3. **Технические требования**

```
- Поддерживай совместимость с современными браузерами
- Используй семантический HTML5
- Всегда добавляй alt атрибуты для изображений
- Используй async/await вместо промисов с .then()
```

### 4. **Доступность (Accessibility)**

```
- Всегда добавляй aria-labels где необходимо
- Используй семантические HTML теги
- Обеспечивай keyboard navigation
- Проверяй контрастность цветов
```

### 5. **Производительность**

```
- Оптимизируй изображения перед использованием
- Используй lazy loading для изображений
- Минимизируй количество DOM запросов
```

### 6. **Специфичные для проекта правила**

```
- Следуй дизайну из Figma файла FD_ 2.fig
- Используй цвета из палитры: #1b1919, #545050, #fff
- Шрифты: 'Fira Sans Condensed' для текста, 'Oswald' для заголовков
```

## Примеры правил

### Простой пример:

```
Используй современный JavaScript (ES6+).
Всегда добавляй обработку ошибок.
Комментируй сложную логику.
```

### Подробный пример:

```
## Стиль кода
- Используй camelCase для переменных и функций
- Используй UPPER_CASE для констант
- Добавляй JSDoc комментарии для функций

## HTML
- Используй семантические теги (article, section, nav)
- Всегда закрывай теги
- Используй атрибут lang="ru" для русского контента

## CSS
- Используй BEM методологию
- Группируй связанные стили
- Используй CSS переменные для цветов

## JavaScript
- Используй const по умолчанию, let только когда нужно переприсваивать
- Избегай глобальных переменных
- Используй стрелочные функции где возможно
```

## Как использовать

1. Создайте файл `.cursorrules` в корне проекта
2. Добавьте правила в свободной форме
3. Сохраните файл
4. Cursor автоматически будет использовать эти правила при работе с AI

## Примечания

- Правила применяются ко всем AI-взаимодействиям в проекте
- Вы можете обновлять правила в любое время
- Правила можно комментировать (используйте # для комментариев)
- Файл `.cursorrules` обычно не коммитится в git (добавьте в .gitignore), если не хотите делиться правилами
